Version Information

Dies ist die Eigenständige Version vom AJAX Chat (blueimp), die für sich alleine läuft, ohne eine weitere Webanwendung. Diese Version eignet sich gut für die Anpassung Ihrer eigenen Integration oder für die Verwendung als eigenständige Version.

Es wird in diesem Paket nur die Eigenständige Version unterstützt.


AJAX steht für "Asynchrones JavaScript und XML".
Der AJAX-Chat-Client (Ihr Browser) verwendet JavaScript, um den Webserver nach Aktualisierungen zu fragen. Anstatt eine komplette HTML-Seite zu liefern, werden nur aktualisierte Daten im XML-Format gesendet. Durch die Verwendung von JavaScript kann die Chatseite aktualisiert werden, ohne dass die gesamte Seite neu geladen werden muss. PHP wird für die Kommunikation mit der Datenbank und die Authentifizierung der Benutzer verwendet.

02. Installation

Laden Sie Ihre bevorzugte Version von AJAX Chat herunter und entpacken Sie die Datei auf Ihrem Computer.

Verwenden Sie einen geeigneten Texteditor!

Um PHP-Dateien zu bearbeiten, benötigen Sie einen guten Texteditor. Sie sollten zum Bearbeiten von PHP-Dateien nicht den Windows-Editor, Wordpad oder Microsoft Word verwenden. Diese Programme fügen den Dateien eine sogenannte Byte-Order-Mark (BOM) hinzu, was dazu führen kann, dass der Chat nicht richtig funktioniert. Wir empfehlen die Verwendung von Notepad ++ (https://notepad-plus-plus.org/) zum Bearbeiten aller Dateien. Es bietet außerdem den Vorteil, dass Ihre Dateien farblich gekennzeichnet werden, sodass Sie sie einfacher bearbeiten können. Wenn Sie eine Fehlermeldung wie „Header-Informationen können nicht geändert werden – Header bereits gesendet“ erhalten, liegt dies wahrscheinlich daran, dass Sie eines der oben genannten Programme zum Bearbeiten von Dateien verwendet haben.

Datenbankeinstellungen konfigurieren

Als erstes und wichtigstes müssen Sie AJAX Chat mitteilen, wie eine Verbindung zu Ihrer Datenbank hergestellt werden soll. Diese und alle Kerneinstellungen müssen sich in der Datei src/config.php befinden. Sie müssen diese Datei erstellen.
Eine Beispieldatei für die Datei config.php finden Sie in src/config.php.example, die im Lieferumfang des Chats enthalten ist.
Duplizieren Sie diese Datei und speichern Sie sie als config.php (oder löschen Sie einfach .example am Ende des Dateinamens) und füllen Sie dann mindestens die folgenden vier Felder in der Datei aus:


$config['dbConnection']['host'] = 'ihr_datenbank_hostname';
$config['dbConnection']['user'] = 'ihr_datenbank_benutzername';
$config['dbConnection']['pass'] = 'ihr_datenbank_passwort';
$config['dbConnection']['name'] = 'ihr_datenbank_name';

Es genügt zu sagen, dass Sie diese Informationen benötigen. Sprechen Sie mit Ihrem Hosting-Anbieter, wenn Sie es nicht wissen.
In den meisten Fällen funktioniert der Chat nur, wenn diese Felder ausgefüllt sind, und Sie können mit dem nächsten Schritt fortfahren.

Wenn Ihr Host MySQL nicht verwendet, müssen Sie das Feld „Verbindungstyp“ ändern:
$config['dbConnection']['type'] = null;
Wenn dies auf „null“ gesetzt ist, wird standardmäßig „mysqli“ verwendet. In den meisten Fällen kann dieses Feld auf „null“ belassen werden.

Sie können auf einen vorhandenen Datenbankverbindungslink oder ein vorhandenes Objekt verweisen, indem Sie Folgendes ändern:
$config['dbConnection']['link'] = null;
Wenn dieser Wert auf „null“ gesetzt ist, wird eine neue Datenbankverbindung erstellt.

Cookies konfigurieren

Als nächstes und wichtigstes müssen Sie die Cookie-Einstellungen tätigen. Sprechen Sie mit Ihrem Hosting-Anbieter, wenn Sie es nicht wissen. Ohne diese Angaben kann der Chat nicht starten.

Beide Konfigurationsdateien müssen bearbeitet werden:

src/config.php


$config['sessionCookiePath'] = '/';
$config['sessionCookieDomain'] = null;
$config['sessionCookieSecure'] = null;
$config['sessioncookieSamesite'] = 'Lax';

public/js/config.js


cookiePath: '/',
cookieDomain: null,
cookieSecure: false,
cookieSamesite: 'Lax',

Wählen Sie Ihre Kanaleinstellungen

Bearbeiten Sie die Datei src/data/channels.json. Wir haben Ihnen zwei Beispielkanäle zur Verfügung gestellt, mit den Namen „Public“ und „Private“. Sie können Ihre eigenen hinzufügen oder sie unverändert lassen. Kanäle folgen dem folgenden Format:


"0": "Public",

Jeder Kanal muss eine eindeutige Kanal-ID-Nummer und einen eindeutigen Namen haben. Leerzeichen in den Kanalnamen werden in den Unterstrich „_“ umgewandelt.

Fügen Sie Ihre Benutzer hinzu

Datenschutz!

Wenn Sie keine Benutzerdefinierte Benutzerverwaltung verwenden, lesen Sie unbedingt nachfolgende Hinweise!

Um ihre Daten zu schützen müssen Sie die Standard-Passwörter der Standard-User ändern.
Ändern Sie die Passwörter ($users[*]['password']) in sichere Passwörter!!!

Die Standalone-Version von Chat verwendet eine PHP-Datei zum Speichern von Benutzern und Räumen, während die Datenbank für Chat-Nachrichten, Einladungen und Sperren verwendet wird. Die Integrationsversionen nutzen typischerweise eine Datenbank für Benutzer und Räume. Wenn Sie Benutzer verwalten möchten, ohne eine PHP-Datei bearbeiten zu müssen, sollten Sie die Verwendung einer Integrationsversion in Betracht ziehen.

Bearbeiten Sie Benutzer in Datei "users.php" im
Verzeichnis src/data/
Benutzer folgen dem folgenden Format:


$users[user id] = array();
$users[user id]['userRole'] = AJAX_CHAT_ROLE;
$users[user id]['userName'] = 'user name';
$users[user id]['password'] = 'user password';
$users[user id]['channels'] = array(allowed channel ids);

Jeder Benutzer muss eine eindeutige Benutzer-ID-Nummer und einen eindeutigen Namen haben. Der erste Benutzer in der Liste (Benutzer-ID 0) wird für die Gastbenutzereinstellungen verwendet. Alle Gastbenutzer haben Zugriff auf die für diesen Benutzer festgelegten Kanäle und die Benutzerrolle „AJAX_CHAT_GUEST“. Registrierte Benutzer können die Benutzerrollen „AJAX_CHAT_USER“, „AJAX_CHAT_MODERATOR“ oder „AJAX_CHAT_ADMIN“ haben. (Groß-/Kleinschreibung beachten, bitte genau eingeben).
Die Liste der Kanäle, auf die ein Benutzer Zugriff hat, kann für jeden Benutzer individuell festgelegt werden. Kanal-IDs werden durch Kommas getrennt. zB: „array(0,1,23);“ ermöglicht die Kanäle 0, 1 und 23. Leerzeichen in den Benutzernamen werden in den Unterstrich „_“ umgewandelt.

Erstellen Sie die Datenbanktabellen

Für die Erstellung der Datenbank stehen Ihnen zwei Möglichkeiten zur Verfügung. Die erste und normalerweise einfachste Möglichkeit besteht darin, das in AJAX Chat enthaltene Installationsskript auszuführen. Alternativ können Sie ein Datenbanktool wie PHPMyAdmin verwenden, um die Tabellen manuell zu erstellen.

  1. Um das Installationsskript zu verwenden, rufen Sie die folgende URL in Ihrem Browser auf:
    https://example.org/pfad/zum/chat/install.php
    Dabei ist „http://example.org/pfad/zum/chat/“ die tatsächliche URL zu Ihrem chat-Verzeichnis.
    Vergessen Sie nicht, die Datei install.php zu löschen, nachdem Sie diesen Schritt abgeschlossen haben!
  2. Um es manuell mit PHPMyAdmin oder einem ähnlichen Tool zu installieren, kopieren Sie den Inhalt der
    src/migrations/chat.sql Datei
    und führen Sie sie als Abfrage aus.

Mit beiden Methoden werden die Tabellen erstellt, die Ihre Datenbank zum Speichern von Chatnachrichten und anderen Informationen benötigt.

Löschen Sie das Installationsskript

Löschen Sie die Datei public/index.php aus dem Chat-Verzeichnis auf Ihrem Server. Sie können auch die Datei src/migrations/chat.sql löschen.

Herzlichen Glückwunsch! Es ist geschafft

Sie sind fertig! Um Ihren Chat zu testen, navigieren Sie in einem Browser zu Ihrer Chat-URL:
http://example.org/pfad/zum/chat/index.php
Sie können den Chat jetzt individuell an Ihre Bedürfnisse anpassen.

03. Configure and customize

Konfigurationsdateien

AJAX Chat ist vollständig anpassbar und enthält zwei Konfigurationsdateien:

  1. src/config.php: Diese Datei enthält die wichtigsten Konfigurationsoptionen für den Chat. Hier finden Sie wesentliche Optionen zur Konfiguration der Datenbank, Sicherheit, verfügbaren Sprachen usw.
  2. public/js/config.js: Diese Datei enthält clientseitige Einstellungen, die die Standardoptionen Ihrer Benutzer im Chat ändern. Viele dieser Einstellungen können vom Benutzer in seinen Optionen geändert werden, einige (wie die Aktualisierungsrate) jedoch nicht.

Beide Dateien sind gut kommentiert und enthalten Informationen zur Bedeutung der Einstellungen.

Anpassen des Layouts

Das Layout von AJAX Chat ist mithilfe von CSS (Cascaded Style Sheets) vollständig anpassbar.
AJAX Chat verfügt über einen vordefinierten Satz von Styles. Gehen Sie wie folgt vor, um Ihren eigenen Style hinzuzufügen:

  1. Fügen Sie eine neue CSS-Datei in public/css (z.B. meinstyle.css) hinzu, indem Sie einen der vorhandenen Style aus dem CSS-Verzeichnis kopieren.
  2. Bearbeiten Sie Ihre Datei (public/css/meinstyle.css) und passen Sie die CSS-Einstellungen nach Ihren Wünschen an.
  3. Fügen Sie den Namen Ihres Style ohne Dateierweiterung zu den verfügbaren Styles in src/config.php hinzu:

    // Available styles:
    $config['styleAvailable'] = array('meinstyle','beige','black','grey');
    // Default style:
    $config['styleDefault'] = 'meinstyle';

Um das Layout weiter anzupassen, können Sie die Template-Dateien in src/template/ anpassen.

Stellen Sie sicher, dass Sie gültiges XHTML erstellen, da sonst in modernen Browsern Fehler auftreten. Dies liegt daran, dass der Seiteninhaltstyp „application/xhtml+xml“ lautet. Die Verwendung dieses Inhaltstyps verbessert die Leistung beim Bearbeiten des Document Object Model (DOM).

Wenn Sie aus irgendeinem Grund kein gültiges XHTML erstellen können, können Sie einen HTML-Inhaltstyp erzwingen. Bearbeiten Sie einfach src/config.php und legen Sie die folgende Option fest:


$config['contentType'] = 'text/html';

Anpassen der Spracheinstellungen

AJAX Chat wird mit zwei Sprachdateiverzeichnissen geliefert:

  1. src/lang/: Dieses Verzeichnis enthält die Sprachdateien, die für die Template-Dateien verwendet werden. Dabei handelt es sich um PHP-Dateien mit der Endung „.php“.
  2. public/js/lang/: Diese Datei enthält clientseitige Einstellungen, die die Standardoptionen Ihrer Benutzer im Chat ändern. Viele dieser Einstellungen können vom Benutzer in seinen Optionen geändert werden, einige (wie die Aktualisierungsrate) jedoch nicht.

Viele Sprachen sind bereits im Download enthalten und können durch Bearbeiten dieser Dateien angepasst werden. Für jede Sprache benötigen Sie in beiden Verzeichnissen eine Datei mit dem Sprachcode als Dateinamen (z.B. de.js und de.php). Der Sprachcode wird gemäß den ISO 639-Standards verwendet.

Die Dateien für die deutsche Lokalisierung (Sprachcode „de“) sind public/js/lang/de.js und src/lang/de.php.

Wenn Sie Ihre eigene Lokalisierung erstellen, müssen Sie die Dateien in die richtigen Ordner ablegen und dann zwei Änderungen an src/config.php vornehmen:

  1. Fügen Sie den Sprachcode hinzu (dieser muss mit dem Dateinamen übereinstimmen, den Sie für die Sprache ausgewählt haben. Denken Sie daran, Kommas korrekt zu verwenden, um mehrere Sprachcodes zu trennen):

    $config['langAvailable'] = array('de');
  2. Fügen Sie den Namen der Sprache hinzu (das sehen Benutzer im Dropdown-Menü zur Auswahl der Sprache):

    $config['langNames'] = array('de'=>'Deutsch');

Um Fehler zu vermeiden, sollten Sie diese Regeln beachten:

  1. Stellen Sie sicher, dass Sie Ihre Lokalisierungsdateien in UTF-8 kodieren (ohne Byte-Order-Markierung).
  2. Verwenden Sie in Ihren Lokalisierungsdateien keine HTML-Entitäten.
  3. Entfernen Sie kein „%s“ in den JavaScript-Sprachdateien – diese sind mit dynamischen Daten gefüllt.

04. Logs

Zugriff auf die Protokolle

Standardmäßig ist im Chat „Online Benutzer“ Menü für die Benutzerrolle „AJAX_CHAT_ADMIN“ ein Link zu den Chat-Protokollen.

Um direkt auf die Protokolle zu zuzugreifen, müssen Sie den GET-Parameter view=logs zu Ihrer Chat-URL hinzufügen (fügen Sie ?view=logs am Ende der URL hinzu):

z.B. http://example.org/pfad/zum/chat/?view=logs

Wenn Sie noch nicht angemeldet sind, müssen Sie sich als Administrator anmelden, um auf die Protokolle zuzugreifen.

Die Protokollansicht ermöglicht es Ihnen, die neuesten Chatnachrichten auf allen Kanälen zu überwachen. Es ist auch möglich, die Protokolle privater Räume und privater Nachrichten einzusehen. Sie haben die Möglichkeit, die Protokolle nach Datum, Uhrzeit und Suchbegriffen zu filtern.

Der Suchfilter akzeptiert reguläre Ausdrücke im MySQL-Stil, wie hier beschrieben:
https://dev.mysql.com/doc/refman/8.0/en/regexp.html
Sie können mit der folgenden Syntax nach IPs suchen: ip=127.0.0.1

05. Shoutbox

AJAX Chat kann auch als Shoutbox verwendet werden – hier ist eine kurze Anleitung zur Einrichtung:

Shoutbox Stylesheet

Fügen Sie dem Stylesheet (CSS) aller Seiten, auf denen die Shoutbox angezeigt wird, die folgende Zeile hinzu:


@import url("http://example.org/pfad/zum/chat/css/shoutbox.css");

Ersetzen Sie http://example.org/pfad/zum/chat/ durch die URL zum Chat. Ändern Sie css/shoutbox.css nach Ihren Wünschen.

Shoutbox Funktion

Fügen Sie Ihrem PHP-Code die folgende Funktion hinzu:


<?php
function getShoutBoxContent()
{
  // Path to the chat directory:
  define('AJAX_CHAT_PATH', dirname($_SERVER['SCRIPT_FILENAME']).'/../');

  // Include Class libraries:
  // if you don't want to use Composer then commented without // the autoloader to vendor and use the bootstrap to bootstrap

  // with Composer
  require(AJAX_CHAT_PATH.'vendor/autoload.php');

  // without Composer. Autoloader is equivalent to PSR-4
  #require(AJAX_CHAT_PATH.'bootstrap/autoload.php');

  $shoutbox = new AjaxChat\Integrations\Standalone\CustomAJAXChatShoutBox();

  return $shoutbox->getShoutBoxContent();
}
?>

Stellen Sie sicher, dass AJAX_CHAT_URL und AJAX_CHAT_PATH auf das Chat-Verzeichnis verweisen.

Shoutbox-Ausgabe

Zeigen Sie den Shoutbox-Inhalt mit der Shoutbox-Funktion an:


<div style="width:200px"><?= getShoutBoxContent(); ?></div>

06. Benutzerdefinierte Integrationsinformationen

Als Beispiel wird eine Integration von phpBB angenommen. Die genauen Informationen von benötigten externen Funktionen erörtern Sie in dem Support Bereich der Software worin Sie den Chat integrieren wollen.
Auf die extern benötigten Funktionen gibt es keine Unterstützung.

  1. Erstellen sie in src/AjaxChat/Integrations/ ein Verzeichnis „Phpbb3“.
  2. Erstellen Sie eine Datei „CustomAJAXChat.php“.
  3. Erstellen Sie eine Datei „CustomAJAXChatShoutBox.php“.

CustomAJAXChat.php ist die Hauptdatei für eine benutzerdefinierte Integration von AJAX Chat. Es enthält die Klasse CustomAJAXChat, die die Klasse AJAXChat erweitert und einige wichtige Methoden überschreibt. Die Klasse AJAXChat enthält die Haupt-Chat-Logik. Eine Instanz der Klasse CustomAJAXChat (die die Haupt-Chat-Logik von AJAXChat erbt) wird verwendet, um den Chat in index.php zu initialisieren:

Verzeichnis src/AjaxChat/Integrations/
  • | - Phpbb3
  • - CustomAJAXChat.php
  • - CustomAJAXChatShoutBox.php

CustomAJAXChat.php ist die Hauptdatei für eine benutzerdefinierte Integration von AJAX Chat. Es enthält die Klasse CustomAJAXChat, die die Klasse AJAXChat erweitert und einige wichtige Methoden überschreibt. Die Klasse AJAXChat enthält die Haupt-Chat-Logik. Eine Instanz der Klasse CustomAJAXChat (die die Haupt-Chat-Logik von AJAXChat erbt) wird verwendet, um den Chat in public/index.php zu initialisieren.

CustomAJAXChatShoutBox.php ist die Hauptdatei für eine benutzerdefinierte Integration als Shoutbox in das bestehenes Template der Webseite.

Um dem Chat mitzuteilen das phpBB integriert ist muss in der Datei src/config.php die Konfiguration von „standalone“ zu „phpbb3“ geändert werden


// The Integrations Directory src/AjaxChat/Integrations/*
// Standard = 'standalone' (* e.g. 'phpbb3', will be integrate with phpBB):
$config['integration'] = 'phpbb3';

Dieses Beispiel ist im Paket enthalten. Es ist nicht überprüft auf Funktion und für die enthaltenen externen Funktionen von phpBB gibt es keine Unterstützung.

Tipp für phpBB: Der Download ist nicht mehr verfügbar, aber es gibt nützliche Hinweise. Mahony's phpBB3.x Support
